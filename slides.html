2 de 5 — fundamentos
El método de mínimos cuadrados
OLS minimiza la suma de errores al cuadrado. La solución viene de una proyección ortogonal sobre el espacio columna de X.
S(β) = (y − Xβ)ᵀ(y − Xβ)
→ dS/dβ = 0 →
β̂ = (XᵀX)⁻¹ Xᵀy

OLSModelo lineal

X = [1, x]. Una sola pendiente para todo el rango. Rápido pero rígido.

B-splineModelo flexible

X = bs(x, df=10). La misma fórmula β̂, pero X es mucho más rica.

# R — mínimos cuadrados manual
X <- cbind(1, x)  # matriz de diseño
beta <- solve(t(X) %*% X) %*% t(X) %*% y
y_hat <- X %*% beta
Clave: lo que cambia entre OLS y splines es la matriz X, no la mecánica algebraica.

5 de 5 — comparación
OLS vs. B-splines: el mismo motor, otra matriz X
Los tres modelos usan β̂ = (XᵀX)⁻¹Xᵀy. Lo que los diferencia es cómo construyen X.

OLSModelo rígido

X = [1, x]. Sesgo alto cuando la relación cambia de tramo.

B-splineModelo flexible

X = bs(x, df=10). Soporte local, estabilidad numérica, control con λ.

Conclusión: los B-splines generalizan la matriz de diseño X para ganar flexibilidad sin abandonar la proyección ortogonal.
